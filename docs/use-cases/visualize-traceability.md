# Visualize the traceability graph

## Summary

Render the product-requirement → requirement → use-case/component
graph as Mermaid or JSON derived from the canonical
traceability graph.

## Owner

Any authoring actor — typically generated by **Author**, **Analyst**, or **Architect** during review, and consumed by **Reviewers** *(role)* and **Stakeholders** *(actor)* to confirm coverage. See [../actors.md](../actors.md) and [../roles.md](../roles.md).

## Purpose

Provide reviewers and stakeholders a visual overview of coverage: which
product requirements have a realising model requirement, which requirements
have implementers, which artifacts are orphaned, and how use cases fan out
into components.

## Entry point

`specs visualize traceability [--format mermaid|json] [--out <path>] [--serve] [--listen <addr>]`

Or VS Code palette: **Specs: Visualize traceability** (renders in a
webview). For a browser-based view, run `specs visualize traceability --serve`
and open the printed local URL. The shared UI defaults to a layered view with
product requirements above requirements, then use cases and implementers, and
can be relaid out as organic or grid. Both UIs can also add a new
traceability edge by choosing a relation kind and two nodes, or remove the
currently selected traceability edge, then persist the updated canonical
relations. The shared UI also shows an inline inspector for the active node,
relation, or pending add-edge selection. Edge additions and removals require
confirmation before the shared UI persists the canonical relation update.

## Exit point

A graph written to `--out` (or stdout) in the requested format, or a local
web server hosting the same Cytoscape-based graph UI. Mermaid is suitable for
review artifacts; JSON is suitable for downstream tooling and UI
consumers. The VS Code panel and the local server both read the canonical
graph JSON. Added or removed edges update the canonical relation part files
through the same engine-backed path.

## Iteration

Regenerate after every CR drain or before review milestones. Pair
with [`specs graph validate`](verify-traceability.md) to confirm the
visual gaps correspond to a real traceability validation failure (or vice versa).
